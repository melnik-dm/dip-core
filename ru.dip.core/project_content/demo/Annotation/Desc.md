Настоящий проект демонстрирует возможности инструмента DIP.

Для именования файлов в проекте может быть использован любой метод. Далее
представлен один из неограниченного количества вариантов именования. Если в
директории несколько текстовых файлов или поддиректорий, то файл в зависимости
от наполнения может называться "Intro.md". Если в директории только один
текстовый файл и нет поддиректорий, то файл должен называться "Desc.md". Если
текстовых файлов в директории несколько, то ни один файл не должен называться
"Desc.md". Имена директорий и файлов именуются с большой буквы.

Отображаемые имена разделов задаются через описание (Edit description).

Каждый объект проекта (директория или файл) имеет уникальный идентификатор.
Идентификатор определяется путем до объекта относительно проекта. При хранении
проекта в системе контроля версий к идентификатору объекта может быть добавлена
его редакция.

При работе в DIP рекомендуется хранить проект в Git.
